## 一、评测协议

| 模型 | 参考三指标（PSNR / SSIM / LPIPS） | VBench |
|---|---|---|
| **LongCat** | 第一个生成帧（frame 93）单帧读数 | ✅ 要 |
| **HY-WorldPlay** | **全部生成视频的逐帧均值**（帧 [1, 末)，189 帧配置） | ✅ 要 |
| **Self-Forcing** | ❌ 不测（无条件前缀，onset 落近无损区无判别力） | ✅ **只测 VBench** |

实现口径（不变）：
- SSIM = paper `metric.py` 原版（11×11 avg_pool 局部窗）；
- LPIPS = paper 口径（[0,1] 直喂 `lpips.LPIPS(net="vgg")`，不归一）；
- 参考 = 与被测**完全同配置**（prompt/seed/pose/长度）的 BF16 run；
- VBench = 逐行复刻的 `imaging_quality`（`vbench_iq.py`，前缀窗口均值）；
- BPE 一律真实现口径（含 zero-point/scale 开销；N4 = 2.3125，QVG = 2.326）。

## 二、研究目标（用户 0717 定稿：三模型同时变好才算完成）

在上述协议下，找到 **BPE 比 QVG（2.326）更低、且指标更好**的方法。
**"比之前好"= 超过 QVG**（N4 是我们自己的方法，不作为门槛；它是当前最佳候选）。
**完成判据 = 以下三条同时满足**（缺一不算完成）：

1. **LongCat 超过 QVG**：f93 三指标超过 QVG（28.73 / 0.9033 / 0.089）
   ——N4 已过此关（31.79 / 0.9424 / 0.067），新方法不应明显低于 N4 这个候选水位；
2. **HY-WorldPlay 超过 QVG**：全程三指标超过 QVG（18.77 / 0.4584 / 0.3740）
   ——N4 现状（18.15 / 0.4637 / 0.3799）未过此关。由于全程均值被断崖支配，
   主要抓手是**推迟或消除回访分岔**（断崖每晚一帧，均值涨一截；INT4 断崖晚 6 帧
   就让全程从 18.8 → 22.5）；
3. **SF 超过 QVG**：VBench 超过 QVG（700f 70.41）——N4 现状 70.26，差 0.15（协议
   噪声级），需反超。

外加硬约束：**BPE 比 QVG 低就行**（真实现口径 < 2.326 即满足，不要求大幅更低
——N4 的 2.3125 已满足；质量优先，不为压 BPE 牺牲指标）。

## 三、方法路线与约束

0a. **【用户 0718 新增】记账对称**：任何压缩技巧（打包/对称格等）必须同样应用于
   QVG 的 BPE 再比较——不许自己骗自己。干净车道：双方按各自自然实现记账
   （QVG 2.326 vs N4 系 2.3125）。trit 打包单方面记 1.6 的做法作废。
0b. **【用户 0718 新增】方法必须简单、本质、普适**：一条规则管所有模型所有 case；
   事件级调度（N19）判定过于复杂，降级为机制证据、不作为交付方法。
0. **【用户 0717 新增硬约束】禁止 k-means / 字典型操作**——k-means 本身开销太贵。
   方法必须走 **PCA + OSCAR（arXiv 2605.17757）思想**的无字典路线，在 INT2 下达标。
   （已达标的 N12 因含 k-means 被此约束排除出最终答案，但其三关数字保留为
   **可达性上界参考**：LC 32.16/0.9438/0.0659、HY 18.841/0.5003/0.3373、SF VBench700 71.52。）
1. **可以以 PCA（N4 框架）为基础**：均值 + 低秩子空间减法 + 非对称残差。
2. **借鉴 OSCAR（arXiv 2605.17757）的思想，不照搬方法**：核心思想 = KV 各维度对
   模型输出的重要性不均等，应给**真正影响注意力输出的维度分配更多比特预算**
   （OSCAR 的实现 = K←QᵀQ、V←注意力加权 SST 的协方差旋转 + 离线校准）。
3. **必须 online**：OSCAR 依赖 offline calibration data，照搬 overhead 不可接受。
   有利观察：**在线重要性信号是免费的**——生成时当前 chunk 的 Q 就在手上（QᵀQ 可
   当场算，无需校准集），近期注意力权重也在手上；N4 本来就逐 chunk 在线，重要性
   统计可搭同一班车。
4. 已知负结果边界（勿重蹈）：把 offline QᵀQ 基直接当 N4 的**减法基**有害
   （r=4 掉 2.9 dB，0715 §结果三）——借"重要性加权预算"，不借"它的基"；
   HY 上加秩（r=8）与半头分裂也已证伪（0717）。
5. 手段：web 检索相关工作（online importance-aware KV quantization）+ 自行设计。

## 五、HY 全程协议的直接含义（设计提示）

全程均值 = (断崖前长度 × 平台值 + 崖后长度 × 地板值) / 总长。三个可动的杠杆：

- **推迟断崖**（最大杠杆）：回访时刻被检索的老 chunk 保真度决定是否分岔——
  对"将来可能被回访的内容"多花比特（重要性 = 未来被检索概率 × 注意力权重）；
- **抬高崖后地板**（INT4 证明可行：0.66 vs 0.37 SSIM）：分岔后的世界一致性；
- **抬平台**（N4 目前在这一项上还输 QVG 3.1 dB，也要补）。
