## 二、新数据点 1：SF 上存在参考三指标（paper 正文没有的）

rebuttal 两处给了 SF 的 vs-BF16 读数：

| 实验 | QVG PSNR/SSIM/LPIPS |
|---|---|
| sink-window 对比（vs 原生滑窗 16.61、SDV2 式 sink+window 21.48） | **27.42 / 0.8729 / 0.0769** |
| centroid cache 消融（iters=6 无缓存 / 20 无缓存 / 6+缓存） | 26.94 / 27.39 / **27.42**（PSNR） |
| "challenging cases"（快速运动/换景/低冗余；模型未注明） | 27.941 / 0.879 / 0.083 |

**关键观察**：作者所有参考指标三元组——LC 28.7/0.909/0.065、HY 29.2/0.882/0.094、
SF 27.4/0.873/0.077、LC@720p 26.0/0.817/0.118、challenging 27.9/0.879/0.083——
全部挤在 **26-29 dB / 0.82-0.91 / 0.06-0.12** 的窄带里，没有任何一处出现混沌地板值
（我们实测：SF 全程 19.7@777f、HY 全程 18.8@189f）。这与"所有测量都落在分岔前区制"
一致，与"长视频全程均值"不相容。

**复现检验（SF 前缀扫描，我们的 195-latent 数组）**：PSNR 前缀均值在 ~160 帧处路过
27.4，但该处 SSIM=0.918、LPIPS=0.175，与他们的 0.873/0.077 **联合对不上**——与 HY
同款结论：**我们的 run 上不存在能联合复现其三元组的帧范围**（他们的 run 分岔行为
必然不同，或另有未披露口径）。

## 三、新数据点 2-4（工程情报）

1. **FP8 E4M3 scale 存储被确认**（rebuttal W5 原话）——N4 impl-plan 的 M0 风险探测
   有了明确对标：QVG 真实现就是 FP8 scale，我们的 BPE 2.3125 账本同一假设。
2. **推荐默认配置**（rebuttal 明说）：K=256、INT2、chunk≈40k tokens、PRQ stages=1 或 4、
   group size=64；k-means 迭代上限 20，开 centroid cache 后 5-6 次迭代 ≈ 20 次质量
   （26.94→27.42 只差 0.5 dB）——我们 kernel-speed 对比里 iters=2 的流式配置比这更激进。
3. **k-means seed 敏感性**：作者只在 VBench IQ 上测（±0.011，≈0.01%级），**没有报
   PSNR 的 seed 方差**——我们 0713 测到的 PSNR σ=0.18/长尾 ±1-2 dB 与此不矛盾
   （VBench 无参考、对采样路径不敏感），我们的方差数据仍是独家。
4. **sink-window 基线被 QVG 大幅击败**（16.6/21.5 vs 27.4）＋ batch 1/2/5 开销稳定
   ~1.7%、RTX 5090 开销 1.9%——将来写 related/system 对比可引用。
