## 检索结论（2026-07-17）

- **MixKVQ**（arXiv 2512.19206）：per-channel 显著性 = E|Q_d|×K 量化难度，三档混精度，
  **online 零校准**，开销 2.17%——直接验证"Q 幅度加权、在线"可行。
- **Block-GTQ**（arXiv 2606.24033）：RoPE 2D 块能量分 = (E‖q‖²+E‖k‖²)/2，
  贪心比特分配 J=Σ s·4^{-b} 可证最优；校准式，但统计量可在线免费获得；
  证实 RoPE 配对粒度。
- OSCAR（2605.17757）：重要性=旋转到 Q/注意力加权谱；离线校准——只借思想。

## 方法 N5：N4 + 在线 Q 能量加权 K 量化

生成时在 q_norm 出口累计 per(层,头,维) E[q²]（零额外前向）；量化 K 时
`s_d=(E[q_d²]/mean)^{α/2}`（几何均值归一），k′=k⊙s，N4 管线作用于 k′，解码除回。
效果 = PCA 基与残差量化器在注意力 logit 加权误差下工作。V 不动。
真实现代价：每(头,chunk) 一个 D 维 fp16 向量（与 mu 同级，BPE +≈0.005，仍<2.326）。
实现：`pca_quant.py`（PCA_QW_ALPHA + provider）、`pca_launcher.py`（LC q_norm 包装）。

## 用户思路记录（0717 晚，无 k-means 新约束下的 idea 清单）

### Idea #1（用户口述，已确认理解一致）：不用 k-means 也能构造出好的"中心点"

> 用户原话大意：k-means 实际上就是找 k 个中心点；但不用 k-means 也能找到效果好的
> 中心点——要用这种思想。

**展开（讨论定稿）**：k-means 的本质交付物 = 一组**密度自适应**的代表点（数据密处
点多）。这个性质可以构造、不必迭代搜索：

1. **变换空间的格点质心**：PCA/KLT 旋转 + 每维按特征值分配比特 = 用乘积格点当
   隐式质心（2^Σb 个，指数级容量）。**已被数据证实**：HY K 侧 KLT 格点 rel-L2
   0.2005 < 真 k-means 256 质心的 0.278——无迭代的构造性质心打赢迭代搜索的自由质心。
2. **密度自适应格点（k-means 精髓的无迭代移植）**：每个旋转维用**分位数格 /
   1D Lloyd-Max**（percentile 一次算完）；更优雅——旋转后 PCA 坐标近高斯（中心
   极限），直接用**正态分位数码本（NF 格，QLoRA NF4 思想）**：固定码本 + 已有
   per-block scale，零额外存储零计算，即"为高斯数据构造的信息论最优质心"。

**落地形式 = N14：KLT 旋转（一次 eigh）+ NF 非均匀格（查常数表）+ 特征值贪心
比特分配（OSCAR 重要性思想）**——三组件全构造性、零迭代、零校准、无 k-means；
BPE 与 N13 同（LC 2.16 / HY 2.27 / SF 2.15，低于 N12 与 QVG）。
一句话："k-means 用迭代找 256 个自由中心；我们用一次旋转 + 高斯分位数把指数级个
中心排布在数据所在的流形上。"

（N13 离线基线已验证：HY K 0.2005 / V 0.4241，LC K 0.1728 / V 0.3671；
集成路径与原型逐位一致。N14 = N13 + NF 格，待用户指令后进 LC 闸门。）

### Idea #2（用户口述）：拿上一个 chunk 的 Q/S 统计，按 OSCAR 协方差思想算出下一个
### chunk 的 KV 哪些重要方向需要保留

**Formulation（讨论定稿）= 流式 OSCAR 加权 KLT**：
- 逐 chunk 生成的时间局部性 ⇒ **上一 chunk 就是免费的在线校准集**（未来 query 的
  最好估计），OSCAR 的离线校准被结构性替代；
- **K 侧**：真实伤害 = E[(qᵀΔk)²] = Δkᵀ M_Q Δk，M_Q=E[qqᵀ]（上一 chunk，免费累计）。
  用 M_Q 当**度量**：白化 + KLT（= cov(K) 与 M_Q 的广义特征值问题），比特按广义
  特征值（能量×重要性）分配——"哪些方向需要保留"的精确数学形式；
- **V 侧**：在线 SST——量化时刻用上一 chunk 的 Q 对本 chunk K 现算一次注意力质量
  （小 matmul，不改模型），V 在注意力加权协方差的特征基里 KLT + 分配；
- **性质**：①白化+旋转折叠为一个 fp8 解码矩阵，存储与 N13 相同、BPE 不变；
  ②与 0715 负结果一致（attention-aware 基属旋转式——这里正是旋转），与 N5 失败
  机制无关（基变换非对角缩放，无 minmax 打架）；③与 Idea #1 正交可组合。
- **合体 = N15：流式 OSCAR 度量 KLT + NF 密度自适应格 + 广义特征值比特分配**
  ——全在线、零校准、零 k-means。

### Idea #3（来源：DeltaQuant, CVPR 2026，`papers/DeltaQuant_CVPR2026.pdf`）：时空
### cube-mean 当免费质心 + 按重要性自适应 cube 预算

> 渊源注意：DeltaQuant 是 **QVG 同一团队网络的姊妹作**（Xi/Yang/Stoica/Keutzer/
> Song Han），把"减局部代表+量化残差"从 KV 侧（k-means 版）搬到了 W/A 侧（cube 版）；
> **cube 版搬回 KV 是他们没做的空位，天然满足无 k-means 约束。**

**机制**：token 按 t×h×w 局部 cube（16–64 个）分组，cube 均值作 core（FP8，
on-the-fly），只量化 delta。其 ablation 三个关键读数：
1. **cube 越小越好（16–32 最优），全局均值直接崩**——局部性即信息，无需迭代搜索
   （与 Idea #1 的"构造性质心"哲学同宗：这里的"质心"= 空间邻域均值，O(N) 白给）；
2. **FP8 core 足够**（专门 ablate 过），开销 3.2%（C=64）；
3. **阶段自适应 cube 分配**：30% 高噪 timestep 用 C=16、其余 C=64，摊销 C=33 不变
   → PSNR +0.5——"重要性加权预算"的已验证实例，映射到 HY 断崖杠杆：**按回访概率/
   近期注意力权重给热 chunk 小 cube/高位宽、冷 chunk 大 cube/低位宽**，摊销 BPE
   不变、断崖后移（与 Idea #2 的重要性度量天然衔接：M_Q 决定"谁热"，cube 粒度执行）。

**诊断印证**：其对 SVDQuant 的批评（激活统计随 timestep 剧烈漂移 → 离线静态变换
在其他步有害）与我们 0715 "offline QᵀQ 基掉 2.9 dB"同一病理——凡借基/旋转必须
逐 chunk 在线；cube-delta 无基，结构性免疫漂移。

**BPE 对账（先算后跑）**：fp8 core 摊销 = 8/C bit/值；直接叠 N4（2.3125）：C=64
→ +0.125 = 2.4375 > 2.326 ✗。必须**预算置换**：① delta 更窄更对称 → 非对称残差改
对称（省 zero-point）；② scale 块变粗；③ 削减低秩预算。

**落地形式 = N16：时空 cube-mean core（fp8）+ 对称残差 ± NF 格**——与 Idea #1
正交可组合（cube-mean 吃局部时空冗余，KLT+NF 吃全局谱结构），与 Idea #2 的重要性
分配在 cube 粒度上执行。LC 闸门最小矩阵：先跑"cube-mean + 对称 INT2 残差、无 PCA"
对账 BPE，再扫 C∈{16,32,64} × core∈{fp8,int8} × 残差对称性，按 eval-protocol §一
读 f93 三指标。
