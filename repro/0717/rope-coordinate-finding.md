# 重大发现：流式视频 KV 没有固定读取坐标系——post-RoPE PCA 量化的不可行性

> 本页回答"为什么不能拿 post-RoPE 的 KV 做 PCA 再量化"，并解释这一管线属性如何
> 决定了可行方法的形状。实验支撑：research-log #19-24（KLT/NF/白化系全灭）、
> #33（注意力输出度量失效）、#34（OSCAR 考古）。

## 一、post-RoPE 量化在 LLM 里为什么可行（OSCAR 的地基）

标准 LLM 的位置是**绝对且一次性**的：token s 的 key 写入时被 R_s 旋转一次，之后
永远以 R_s·k 的形态被所有 query 裸点积读取——**每个 key 有唯一的、永不再变的
post-RoPE 形态**。因此可以缓存 post-RoPE key，在这个固定坐标系里做 PCA/旋转/
各向异性误差整形，误差方向与读取度量永久对齐。KIVI、OSCAR（其校准脚本自注
"Dump post-RoPE Q/K/V"）都建立于此。

## 二、三个视频管线里这个前提不存在

**同一个缓存 key 在不同读取时刻会被施加不同的旋转**，没有唯一 post-RoPE 形态可存：

- **HY**（相对 RoPE + FOV 记忆检索 + PRoPE）：被检索的老帧按"相对当前帧的位置"
  重新排位进上下文，rope 按新相对索引现算；prope 还要按**当前相机**做投影变换
  ——同一帧的 key，chunk 5 读是一种旋转、chunk 9 回访读是另一种；
- **LC**（分段续写）：每段生成重新锚定窗口，同一 key 在不同段的有效位置不同，
  `rope_3d` 每次前向按当前 grid 现算；
- **SF**（滚动窗）：位置随窗口滚动重排。

这正是 QVG 论文写明 "**pre-RoPE key caching**" 的原因——流式视频的动态位置语义
**要求**缓存位置无关形态、读取时现转。

## 三、"存写入时刻的 post-RoPE、读取时转差量"也不行（要害）

写入按位置 s 存 R_s·(k+ε)，读取需要 R_s'·k，于是反转再重转：
得 R_s'·k + **R_s'R_s⁻¹·ε**。旋转是等距变换——误差**大小**不变（不会爆炸），
但误差**方向**被随读取时刻变化的 R_s'R_s⁻¹ 搅拌——**写入坐标系里 PCA 精心对齐的
各向异性（重要方向误差小）被彻底打乱**，而对齐正是 PCA 量化的全部价值。

## 四、由此导出的可行性定理（本轮 15+ 机制族证伪的统一解释）

**在读取坐标系不固定的管线里：**

1. **各向异性误差整形（任何固定基上的 KLT/NF 格/白化/按维比特）不可行**——
   张量误差砍半、生成全崩（5 次张量-生成脱钩实录：N8、N14×3、V-KLT）；
2. **各向同性误差的方案不受影响**——各向同性怎么转都是各向同性：QuaRot 的
   Hadamard+均匀 2-bit 能活（LC 30.38）、我们的均匀残差格能活；
3. **PCA 的唯一安全用法是"减法"**——减掉的均值/低秩部分被精确重建（fp 系数
   逐 token 恢复），误差只留在**均匀格残差**里 = 各向同性。Budget-PCA 的全部
   自由度（秩多少、往哪个半区放）都在调"减多少"，从不调"误差往哪个方向放"；
4. **时间/token 轴的各向异性是读取端唯一搅不乱的**——旋转混合的是维度，不混合
   "哪个 chunk"（N19 锚点调度有效的原因；被简单性约束排除但机制成立）。

## 五、对相关工作的定位含义

- OSCAR 的协方差思想没错，但其成立前提（post-RoPE 固定坐标系）是 LLM 特权——
  搬到流式视频需要先回答"读取坐标系在哪"，这正是本发现填的空；
- 把 LLM KV 量化方法（依赖旋转/各向异性的一大族）搬到视频 AR 管线前，先检查
  该管线的位置语义是否动态——**动态则整族不适用**，这是一个可复用的判据。
