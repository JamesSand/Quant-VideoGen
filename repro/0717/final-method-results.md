# 最终方法 Budget-PCA：三模型全超 QVG、BPE 更低、公平记账、零 k-means（定稿）

> **方法一句话**：**逐 chunk 在线 PCA 减法（mean + top-r 自协方差基，一次 eigh、无迭代
> 无校准）+ 分块低位宽残差**；全部配置由一条规则给出——"在 BPE < QVG 的预算内取
> 封顶配置"。与 QVG 同标准的普适性（QVG 自己也逐模型调 iters/K/块参数），
> 但**无 k-means、无字典、无调度**。记账完全对称（三值格与 QVG 同按 2-bit 计，
> 无打包把戏）。

## 二、各模型的封顶配置（一条规则的三个解）

- **LC**（128 维单空间，单量化事件）：r=4，残差非对称 2-bit B=128；
- **HY**（256 维 = rope‖prope 打包）：**半区秩重分配 K=9:0、V=8:1**（总秩 9 = 预算
  封顶；prope 半区被相机变换缠绕、数据价值低——秩归零、仅均值+残差），残差
  asym B=128。备选 K=V=9:0 → 18.824/0.4907/0.3495 同样过线；
- **SF**（128 维，VBench 无参考评测）：r=4，残差**对称三值 B=64（QVG 同款格、同款
  2-bit 记账）**。

BPE 逐项（真实现口径）：残差 2 + scale(/zp) fp8 + coef((Σr×2+8·半区数)/D) + mu/基摊销。
LC 2.3125 / HY 2.320 / SF 2.26，全部 < QVG 2.326（其自身实现口径）。

## 三、为什么是这一形态（关键机制结论，探索全链见 research-log.md）

1. **HY 的解 = prope 半区秩归零**：诊断确认 prope 对 K、V 都做相机相关变换
   （无固定读取度量），其半区子空间数据价值低；把系数预算全部让给 rope 半区，
   HY 从 18.15 → 18.90（梯度 4:4→6:2→7:1→8:1→9:0 单调验证，反向 2:6 更差）；
2. **旋转/变换编码在 KV 上不可行**（本轮最大负结果）：K 是 pre-RoPE 缓存、读取时
   位置旋转混合维度对——任何固定基上的各向异性误差整形（KLT/NF/白化/按维比特）
   张量误差砍半但生成全崩（5 次张量-生成脱钩实录）；OSCAR 不受此困是因为它量化
   post-RoPE K（相对 RoPE 下该前提不存在）；
3. **W_O 读取度量**（N20）是 LC 特效药（32.91 历史新高）但不普适（SF 长时程有害），
   记录为可选增强而非方法本体；
4. **格子是数据依赖的**：LC/HY 吃 4 电平非对称、SF 的 VBench 吃三值细 scale——
   与 QVG 逐模型调参同性质。

## 五、诚实边界

- n=1 单 prompt/seed（QVG 靶同源同限）；HY 的 +0.13 dB 领先在 ±0.2 混沌噪声内，
  SSIM/LPIPS 的领先（+0.029/−0.025）超出噪声——多 prompt 复验仍是定案前提；
- fake-quant 记账；kernel 化待做（管线组件均为 GEMM/逐元素，无迭代）；
- 半区秩规则目前只在 HY（唯一的 packed 模型）上取非平凡值，LC/SF 自动退化为普通 r。
