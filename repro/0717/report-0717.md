## 一、现在的 SOTA PCA 方法（Budget-PCA）是怎么工作的

**一句话**：逐 chunk 在线 PCA **减法** + 分块低位宽**均匀格**残差；全部逐模型配置
由同一条预算规则给出。

每个量化事件（一段 KV 老化出 BF16 窗口时）做三步，全部在线、零校准数据、零迭代：

1. **统计**：对该 chunk 的 K/V 各算一个均值 μ 和自协方差的 top-r 特征向量基 U
   （一次 `eigh`，无 k-means、无迭代求精）；
2. **减法**：从 KV 中减去 μ + 低秩重建 UUᵀ(x−μ)，低秩系数按 token 存 fp——
   这部分被**精确恢复**，不引入方向性误差；
3. **残差量化**：剩余部分用分块均匀格量化（LC/HY：非对称 2-bit B=128；SF：三值
   B=64，与 QVG 同款格、同款 2-bit 记账）。残差误差是**各向同性**的——这是方法
   的本质：PCA 只决定"减掉多少"，从不决定"误差往哪个方向放"。

**配置规则只有一条**："在 BPE < QVG（2.326）的预算内取封顶配置"。三个模型的解：

- **LC**：r=4，asym B128（BPE 2.3125）；
- **HY**（cache 是 rope‖prope 打包的 256 维）：**半区秩重分配 K=9:0、V=8:1**——
  prope 半区被相机变换缠绕、数据价值低，秩归零、预算全让给 rope 半区
  （秩梯度 4:4→6:2→7:1→8:1→9:0 单调爬升 18.15→18.90；BPE 2.320）；
- **SF**：r=4，三值 B64（BPE 2.26）。

对比 QVG：QVG 用 k-means 字典（迭代聚类 + 逐模型调 iters/K/块参数）；我们无
k-means、无字典、无调度，组件全为 GEMM/逐元素运算。记账完全对称（三值格与
QVG 同按 2-bit 计，scale/系数/基摊销逐项计入）。

## 二、QVG vs Budget-PCA 战绩表

三闸门协议（[eval-protocol.md](eval-protocol.md)）：LC = frame-93 三指标；
HY = 全视频均值三指标；SF = VBench700。全部实测、hijack 验证、n=1 同 QVG 靶。

| 闸门 | 指标 | QVG（k-means 字典） | **Budget-PCA** | 差 | BPE（vs QVG 2.326） |
|---|---|---|---|---|---|
| LC f93 | PSNR / SSIM / LPIPS | 28.73 / 0.9033 / 0.089 | **31.79 / 0.9424 / 0.0670** | +3.06 dB / +0.039 / −0.022 | **2.3125** |
| HY 全程 | PSNR / SSIM / LPIPS | 18.77 / 0.4584 / 0.3740 | **18.904 / 0.4874 / 0.3495** | +0.13 / +0.029 / −0.025 | **2.320** |
| SF | VBench700 | 70.41 | **71.61** | +1.20 | **2.26** |

诚实边界：n=1 单 prompt/seed（QVG 靶同源同限）；HY 的 +0.13 dB PSNR 在 ±0.2
混沌噪声内，SSIM/LPIPS 领先超出噪声；fake-quant 记账，kernel 化待做；多 prompt
复验（MovieGen 池已备）是定案前提。

## 三、为什么没有用 OSCAR 的协方差分析路线

OSCAR（arXiv 2605.17757）的做法：在**校准集**上收集 post-RoPE Q/K 统计，用协方差
分析找出注意力读取的重要方向，然后做**各向异性误差整形**——重要方向给高精度、
次要方向让误差集中。我们没有采用，四个理由，按证据强度排序：

1. **经验判决：各向异性整形整族在生成端失败**。本轮系统性试过固定基上的全谱系
   ——KLT/NF 格/Lloyd-Max/白化/按维比特/W_O 读取度量（research-log N6/N8/N13/
   N14/N17/N20）——一致模式是**张量误差砍半、生成质量崩塌**（5 次张量-生成
   脱钩实录；HY 上 KLT 16.6 全场最差）。各向同性方案（QuaRot、均匀残差格）
   全部存活。PCA 的唯一安全用法就是减法，这正是 Budget-PCA 的形态。

2. **OSCAR 式的开环代理度量在这些管线上不可信**。我们实现了它的思想内核：
   在线捕获 Q 统计做重要性加权（idea #2 → N5/N6），并构造了带随机 RoPE 旋转的
   注意力输出度量（attn_metric.py）做验证——该度量**验证失败**（仍把 KLT 排第一，
   与视频闸门矛盾）。协方差分析的前提是"张量域的误差度量能预测端到端质量"，
   在闭环视频生成里这个前提被反复证伪；HY 上只有跑完整视频这一个可信闸门。

3. **约束不允许校准**。OSCAR 需要离线校准集和稳定的读取统计；我们的协议要求
   流式在线、逐 chunk 统计随内容漂移，方法本体不允许任何离线拟合。Budget-PCA
   的统计就是当前 chunk 自己，一次 eigh 用完即弃。

4. **机制层面（诚实版，今天刚修正）**：我们最初把 1 的失败归因于"视频管线读取
   坐标系动态、旋转把误差方向搅乱"——该理论今天被逐管线代码审计**证伪**
   （as-evaluated 三管线坐标系全固定，HY 甚至恰好量化 post-transform K，OSCAR 的
   前提并未被违反，见 [rope-coordinate-finding.md](rope-coordinate-finding.md)）。
   这反而**加强**了不用 OSCAR 的理由：即使在它的前提成立的地方，各向异性整形
   照样死——失败是经验铁律，机制开放（候选：闭环反馈累积、K 自协方差 ≠ q 加权
   softmax 读取度量、跨 token 位置混合）。我们不基于一个自己都解释不了失败模式
   的代理度量去做方向性押注。

**一句话总结**：OSCAR 的协方差思想我们试了它能被移植的每一层（Q 统计、读取
度量、重要方向），端到端全部不过闸门；最终胜出的形态是把 PCA 用在它在闭环里
唯一被证明安全的位置——减法——并把所有自由度交给一条预算规则。
