## 〇、先把两个方法说成同一种语言(这是分析的地基)

两个方法其实是**同一骨架的两种"减法"**:

- QVG:残差 = X − **最近质心**(每 chunk 用 k-means 学 K=256 个质心的字典,
  逐 64 维块查表)→ 残差 2-bit 均匀格;
- 我们:残差 = X − **μ − top-r 线性子空间投影**(每 chunk 一次 eigh)→ 残差
  2-bit 均匀格(通道轴)。

所以问题精确化为:**对视频 KV 数据,"字典减法(256 个点)"和"仿射子空间减法
(r=4 的连续平面)"谁的每比特元数据消掉的能量多?** 外加第二自由度:残差格的
**轴**(QVG token 轴 vs 我们 channel 轴)。分析就沿这两个自由度展开。

## 一、四个假说与判据(方法论主体)

### H1【表示假说】KV 行云是"低秩仿射薄饼",不是"256 个团块"

- **预测**:同一 chunk 上,rank-4 子空间解释的能量 ≫ 256 质心字典消掉的能量
  (按同等元数据比特对齐:r=4 系数 ≈ 每 token 8+16 bit vs 质心索引 8 bit/块+码本摊销);
- **实验**:dump 三管线真实 chunk(复用 kernel-plan 的同一批 npz);画
  (a) **奇异值谱**(衰减越陡越支持薄饼);(b) **"减法效率"曲线**——横轴元数据
  bits/elem,纵轴残差能量占比,两条线:rank r=1..16 扫 vs K=16..4096 扫(k-means
  用 QVG 原装实现跑到收敛);两线的相对位置就是判决;
- **证伪条件**:若 K=256 曲线在同比特下低于 r=4 点,H1 死,得另找解释。

### H2【几何假说】k-means 的欧氏度量被高方差通道劫持,通道轴 scale 是解药

- **预测**:KV 通道方差跨 2-3 个数量级(KIVI 在 LC 33.5 的根源);k-means 的
  质心分配几乎只由前几个大方差通道决定 → 小方差通道的结构完全没被字典编码,
  全部压力落到 token 轴残差格上,而 token 轴格的 scale 又被同 block 里的大方差
  通道撑大 → 小通道被淹没;
- **实验**:(a) 逐通道方差谱(log 轴,三模型);(b) **逐通道残差误差对比图**:
  同 chunk 分别过 QVG 全流程与我们全流程,画每通道的量化误差/信号比——预测
  QVG 在小方差通道上误差比异常大;(c) 消融:把 QVG 的 kmeans 换成"通道白化后
  再 kmeans"(诊断臂,不入正式方法),若显著变好则坐实度量劫持;
- **证伪条件**:QVG 的逐通道误差比均匀 → H2 死。

### H3【预算假说】k-means 的元数据花在了"密度"上,而 KV 需要的是"方向"

- **预测**:K=256 的码本 + 索引开销(QVG 账本里 Indices+Codebook 占 BPE 的
  ~0.2)买到的是对**密集团块**的精细刻画;但 KV 的 token 变化主要发生在少数
  几个连续方向上(H1 的薄饼),字典点数再多也是在薄饼上撒离散点——维度诅咒:
  256 个点覆盖 4 维连续流形的量化误差 ~K^(-1/4),而线性投影一步到位;
- **实验**:2D/3D 可视化 case study——取真实 chunk 的 top-2/top-3 主成分平面,
  散点画 token 云 + 叠 QVG 的 256 质心位置 + 我们的 r=4 平面投影残差;肉眼可见
  "质心稀稀拉拉铺不满连续云"即成立;配数值:同比特下两法在这个子空间内的
  量化 SNR;
- **文献锚**:率失真理论的经典结论(transform coding vs VQ:高斯源下 VQ 需要
  指数级码本才追平变换编码,Goyal 2001《Theoretical Foundations of Transform
  Coding》一系),执行时检索并引用(候选检索词见 §三)。

### H4【工程假说】k-means 自身的收敛与非确定性是次要因素(要排除的干扰项)

- **预测**:LC 用 iters=100(充分收敛)仍输 7dB → 失败是表示问题不是优化问题;
  SF/HY 的 iters=2 + atomic_add 非确定(σ≈0.18)只贡献小量;
- **实验**:iters ∈ {2,10,100} 的 QVG 质量曲线(10 prompt 子集)——若曲线平坦,
  H4 成立(k-means 输不是没跑够),分析可聚焦 H1-H3;
- 这个假说是给分析上保险:防止审稿人一句"你们没把 kmeans 调好"。
