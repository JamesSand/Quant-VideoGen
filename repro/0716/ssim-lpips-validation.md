# SSIM / LPIPS：首帧口径验证 + PCA-KV N4 的全指标检验

任务链：①实现/复用 SSIM、LPIPS 首帧评测 → ②验证与 paper Table 1 能否对上 → ③检验
PCA-KV N4（31.79 PSNR）在这两个指标上是否同样占优。评测帧 = frame 93（首生成帧），
参考 = `results/longcat/bf16/1-0/segment_1.mp4`。

## 1. 口径验证（LongCat INT2，我们 frame-93 vs paper Table 1）

| 方法 | PSNR 我们/paper | SSIM 我们/paper | LPIPS官方口径 我们/paper |
|---|---:|---:|---:|
| RTN | 21.85 / 20.87 | 0.742 / 0.719 | 0.226 / 0.203 |
| QuaRot(≈对称) | 21.42 / 21.57 | 0.740 / 0.759 | 0.202 / 0.171 |
| QVG-Pro | 31.03 / 30.38 | **0.931 / 0.935** | 0.069 / 0.048 |
| QVG | 28.72 / 28.72 | **0.904 / 0.909** | 0.091 / 0.065 |

**结论**：
- **SSIM：首帧口径成立**（QVG/QVG-Pro 差 ≤0.005，基线 ≤0.02）——PSNR 的首帧结论推广成功
- **LPIPS：两段式发现**
  1. 官方 `metric.py` 把 **[0,1] 帧直接喂给 lpips**（未按库约定归一到 [−1,1]，
     `metric.py:131`）——按此"官方口径"复算后差距从 ~2× 缩到 ~0.02-0.03
  2. 残余偏差（排序一致、绝对值系统略高）未能归零，疑似 lpips 库/权重版本或分辨率
     处理差异；**同一代码下的相对比较不受影响**
- 排除项：paper 那句 "report the number of the first generated chunk" 不是 20 帧均值
  （按 93-112 均值算 QVG 只有 17.9，差得远）——就是首帧口径

## 2. PCA-KV N4 的全指标检验（frame-93，同一套代码，公平相对比较）

| 配置 | BPE | PSNR | SSIM ↑ | LPIPS官方 ↓ | LPIPS规范 ↓ |
|---|---:|---:|---:|---:|---:|
| **PCA-KV N4**（r=4 K/V，非对称残差 B128） | **2.253** | **31.79** | **0.942** | **0.067** | **0.110** |
| QVG B64（n=2 独立运行） | 2.326 | 28.97±0.16 | 0.907±0.002 | 0.089±0.001 | 0.136±0.000 |
| QVG-Pro（对照） | 3.30 | 31.03 | 0.931 | 0.069 | 0.113 |

**结论：N4 的 PSNR 优势完整传导到 SSIM 和 LPIPS**——
- vs QVG（同预算档）：SSIM +0.035、LPIPS 好 25%（0.067 vs 0.089）
- vs QVG-Pro（贵 47%）：三项仍全部小胜（SSIM +0.011、LPIPS 0.067 vs 0.069）
- 排除了"PSNR 高但感知质量差（过度平滑）"的担忧：LPIPS 是感知指标，N4 同样最优

## 3. 方法与复现

- SSIM：`precompute_arrays.calc_ssim`（滑窗版，与官方实现同族）
- LPIPS：`lpips.LPIPS(net='vgg')`，双口径并报——官方口径（[0,1] 直喂，对齐 paper）
  与规范口径（×2−1，学术标准）；结论在两个口径下一致
- 一次性脚本见本文档 git 提交中的命令记录；逐帧数组沉淀在 `repro/backup/protosearch/*.npz`

## 口径决策（2026-07-16 定稿）

**本项目此后所有 LPIPS 一律采用 paper 口径**：帧值 [0,1] 直接喂 `lpips.LPIPS(net="vgg")`，
不做库约定的 [−1,1] 归一（复刻官方 `metric.py:131` 的行为）。理由：与 paper 数字可直接
对表；同一刻度下的相对比较不受影响。**代价（须知）**：绝对值与其他论文的标准 LPIPS
不可横比；对外报告时注明"LPIPS (QVG protocol)"。工具已同步：`precompute_arrays.py`
改为此口径（2026-07-16 前生成的 npz 里 lpips 数组是 [−1,1] 归一版，勿混用）。
